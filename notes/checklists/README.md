# Checklists

Step-by-step actionable plans. Each file is a self-contained checklist for a specific goal.

## Format

- One checklist per file, named `<slug>.md` (lowercase, hyphens)
- Steps use `- [ ]` (pending) and `- [x]` (done)
- See `.github/agents/planner.agent.md` § Checklists for full format spec

## Review workflow

When a checklist's completed steps describe live infrastructure not yet in the wiki, the Planner agent will prompt the user to hand off to the Wiki Editor for documentation.
